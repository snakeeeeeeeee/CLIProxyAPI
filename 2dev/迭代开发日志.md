## 2026-07-12：Claude Code 多账号池、API Key 与标准计费

- SQLite 新增严格隔离的账号池、池 API Key 和不可变模型价格版本；内置 `default` 池的 ID 与名称固定为 `default`，不可删除、归档或改名，现有账号、用量和调度事件迁移后归入该池。
- 一个账号只属于一个池，一个 `sk-cap-...` Key 只绑定一个池。请求继续使用 `/claude-acc-pool/v1/*`，服务端根据 Key 确定池，客户端不能通过 Header、Query 或 body 覆盖；空池不会回退到其他池。
- 新 Key 使用 256 位随机熵，SQLite 保存鉴权用 SHA-256、可见前缀和管理界面按需读取的完整值。列表接口不返回完整值；仅接受 `Authorization: Bearer` 或 `x-api-key`，不能用于普通 `/v1/*` 或 Management API；旧 `config.yaml` Key 固定映射到 `default`。
- 会话亲和、活跃会话、prefix lane、冷却和容量统一使用 `claude-acc-pool/<pool_id>` namespace。同一个 Session ID 在不同池中不会共享绑定、候补或主账号提升。
- 账号支持在无在途请求时显式移动。移动会保留 OAuth、代理与健康状态，清除旧池亲和和活跃会话；历史 usage 仍归原池。
- 用量账本按真实上游 attempt 保存池、Key、价格 Revision、缓存写入 5m/1h 和估算成本。下游请求数按 request ID 去重，重试与换号成本保留；纯净模式只改变下游 usage，不影响计费。
- 内置 Sonnet、Opus 和 Haiku 标准价格；Fable 在未配置官方价格前保持“未计价”。价格修改创建不可变 Revision，历史账单不随新价格变化。
- 资源控制台重构为“总览 / 账号池 / API Keys / 代理 IP / 模型与价格 / 系统设置”，支持池详情深链、池级统计、账号 30 天用量、Key 查看复制、价格编辑、账号移动和 `24h/7d/30d/全部` 筛选。
- 修复真实旧单池 SQLite 升级时多池索引早于 `pool_id` 列创建的问题；迁移现在先补列、再建索引并回填 `default`，并增加旧账号/事件/账本表回归测试。
- 完整 Go 测试、后端构建、前端 type-check/build 和 diff 检查通过；新增跨池选号、Key 生命周期、计价快照、迁移及事件时间窗口测试。

## 2026-07-12：账号测试响应与模型额度兼容

- 账号测试兼容标准 JSON、嵌套 `message.content`、SSE `text_delta` 以及旧 `completion/output_text` 响应。
- HTTP 200 但没有文本时不再静默返回空 `reply`，而是返回脱敏响应摘要，例如 `stop_reason=refusal`。
- 成功的账号测试会合并 `anthropic-ratelimit-unified-*` Headers，用于更新模型额度快照。
- OAuth usage 支持新的 `limits[].kind=weekly_scoped` 格式，并从模型 scope 识别 Sonnet、Opus 和 Fable 周额度。
- 控制台固定展示 `7d S / 7d O / 7d F`；未返回独立 Sonnet/Opus 窗口时显示“共享”，不伪造百分比。
- unified Header 的 `overage` 表示额外用量资格，不再误判为模型额度耗尽或影响模型调度。
- 本地真实 smoke：Sonnet 4.6 与 Opus 4.8 正常返回文本；Fable 返回 HTTP 200 + `stop_reason=refusal`，界面能正确解释该结果。
- Anthropic 公开套餐文档未定义“Fable 与 Opus 固定共享 50%”规则；Pro 描述为一个全模型周窗口，Max 描述为全模型周窗口加 Sonnet 窗口。
- OAuth usage 的非活跃 `weekly_scoped` 条目不再显示为虚假的 100%；只有 `is_active=true`、带 reset，或新鲜的 `7d_oi` 响应 Header 才展示独立 Fable 窗口。
- 管理页面的账号测试会写入 usage ledger，记录真实状态码和 usage token，因此会进入 `1h 可用性`、用量统计和 SSE 刷新。
