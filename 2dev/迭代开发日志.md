# 迭代开发日志

## 2026-06-17：Claude Code Account Pool 观测与 Trace 工具增强

### 本次提交

- 提交：`af7d2f99 增强 Claude Code 账号池观测与 Trace 工具`
- 范围：Claude Code Account Pool、资源控制台、代理池联动、运行观测、日志、Trace 对齐工具链。

### 账号池观测增强

- 新增 Claude Code Account Pool 专属运行指标。
- 顶部指标改为账号池自己的数据源，不再混用旧 Claude API Pool。
- 指标包含：
  - 可用账号、账号总数。
  - 全局并发、RPM。
  - 请求数、成功数、失败数、成功率。
  - 真实输入 tokens、真实输出 tokens、缓存创建 tokens、缓存读取 tokens、真实总 tokens。
  - 最近错误、本地拒绝摘要。
- 账号卡片容量展示拆分为：
  - 并发：`in_flight / concurrency_limit`
  - RPM：`rpm_used / rpm_limit`
  - Sticky buffer：`buffer_used / sticky_buffer`

### 账号池专属日志

- 新增 JSONL 日志：`acc-pool-logs/account-pool.log`。
- 默认开启，等级 `info`。
- 默认单文件 `50MB`，保留 `3` 个轮转文件。
- 默认脱敏账号、auth id、session、API key、代理地址等敏感信息。
- 日志记录：
  - 请求选号。
  - 请求成功。
  - 上游错误。
  - 本地拒绝。
  - 账号结果更新。
  - usage 账本写入成功或失败。

### 调度事件

- 补齐 `claude_code_routing_events` 写入。
- 记录事件包括：
  - `selected`
  - `success`
  - `upstream_error`
  - `rejected`
- 记录字段包括：
  - request id
  - 模型、请求模型
  - 账号、代理
  - sticky/session
  - 并发、RPM、容量
  - decision、reason、status code、error
- 本地拒绝会进入日志和调度事件，但不会进入计费用量账本。

### 真实用量统计

- usage ledger 增加真实 token 字段：
  - `raw_input_tokens`
  - `raw_total_tokens`
- 如果开启“纯净 input_tokens”，对外返回可以扣减系统提示词口径，但真实用量统计仍保留 Anthropic 原始 usage。
- UI 运行指标展示真实 token 用量。
- 用量账本合并到运行指标区域，不再作为配置页里的独立区域。

### SSE 实时刷新

- 复用资源池 EventSource。
- usage ledger 写入后推送统计刷新。
- routing event 写入后推送统计刷新。
- 日志配置变更后推送配置刷新。
- UI 收到事件后刷新：
  - 运行指标
  - usage
  - routing events
  - logs
  - accounts
- 一次请求可能触发多次刷新，因为 usage、账号结果、routing event 是分开写入的。

### 资源控制台 UI

- Claude Code 账号池页面增加：
  - 运行指标区。
  - 真实 token 用量区。
  - 最近错误区。
  - 账号卡片容量拆分。
  - 账号池日志面板。
  - 最近调度事件表。
- 日志面板支持：
  - 启用/禁用日志。
  - 调整日志等级。
  - 调整单文件大小。
  - 调整保留文件数。
  - 启用/禁用脱敏。
  - 清空日志。
  - 下载当前日志。
- 日志下载使用 `X-Management-Key` 请求，不把管理密钥放到 URL query。

### Claude Code Trace 工具链

- 新增真实 Claude Code trace recorder：
  - `cmd/claude_trace_recorder`
- 新增 trace diff 工具：
  - `cmd/claude_trace_diff`
- 新增 trace 结构、脱敏、存储、diff、报告能力：
  - `internal/claudetrace`
- 新增账号池 outbound dump：
  - `internal/runtime/executor/claude_trace_dump.go`
- Trace 工具用于对齐真实 Claude Code 请求形态，不自动修改生产 profile。
- `traces/` 已加入 `.gitignore`，真实抓包产物默认不提交。

### Profile 基线参考

- 支持从 Phistory 拉取 Claude Code profile 快照。
- 支持当前运行 profile 与快照做对比。
- 对比仅作为参考，不会直接应用到正常业务请求。
- 避免把 Phistory 的完整超长提示词直接用于账号池生产请求。
